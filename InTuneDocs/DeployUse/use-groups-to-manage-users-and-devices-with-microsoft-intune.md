# 使用 Microsoft Intune 建立群組以管理使用者和裝置

若要建立和管理群組，請使用 Microsoft Intune 管理主控台中的 [群組] 工作區。 [群組概觀]  頁面包含狀態摘要，可幫助您識別並優先處理需要注意的問題：

-   警示
-   軟體更新
-   Endpoint Protection
-   原則
-   軟體管理

此外，群組階層會與狀態摘要一同顯示，以協助您識別及解決所選群組之成員的問題。


> [!TIP]
> 當您建立群組時，請考慮如何套用原則。 例如，您可能會有裝置作業系統特定的原則，以及組織中不同角色特定的原則，或是您已經定義於 Active Directory 中組織單位特定的原則。 有些人認為具有 iOS、Android 及 Windows 特定的裝置群組，以及適用於每個組織角色的使用者群組是非常實用的。
>
> 您可能想要建立要套用到所有群組和裝置的預設原則，以建立公司的基本相容性需求。 接著，針對範圍最廣泛的使用者和裝置類別建立更具體的原則，例如，適用於每個裝置作業系統的電子郵件原則。
>
> 請謹慎為原則命名，以便日後能夠輕鬆地找到它們。 例如，**適用於整家公司的 WP 電子郵件原則**就是一個很好的描述性原則名稱
>
> 當您每次建立嚴格的原則時，會想要將它傳達給您的使用者，如此一來，在您建立更通用的群組和原則之後，就能留意如何建立較小的群組，使您能夠減少不必要的通訊。


## 建立裝置群組

1.  在 Intune 管理主控台中，選擇 [群組]  &gt;  [概觀]  &gt;  [建立群組]。

2.  提供群組的名稱和選擇性描述，並選取一個裝置群組做為父群組。 選擇 [下一步]

3.  在 [定義成員資格準則]  頁面上，選取群組要包含的裝置類型。 設定群組的其他選項取決於您選取的裝置類型：

    -   **電腦：** 指定是否要包含父群組的所有成員、您要包含或排除的組織單位 (OU)，以及您要包含或排除的網域。 電腦的 OU 和網域資訊可透過清查取得。

    -   **行動：**指定只包含受 Intune 管理的行動裝置，受 Exchange ActiveSync 管理的行動裝置，或同時包含兩者。

    -   **所有裝置：** 這個選項會包含所有裝置，不會有以準則為基礎的排除項目。

4.  在 [定義直接成員資格]  頁面上，按一下 [瀏覽] 以包含或排除您指定的個別裝置。 如果您使用此選項選取的裝置不在指定的父群組中，這些裝置會自動新增到父群組。


5.  在 [摘要] 頁面上，檢閱將要執行的動作。 選擇 [完成]

您可以在 [群組] 工作區之父群組下的 [群組] 清單中，找到新建立的群組。 您也可以從這裡編輯或刪除該群組。

## 建立使用者群組

1.  在 Intune 管理主控台中，選擇 [群組]  &gt;  [概觀]  &gt;  [建立群組]。

2.  提供群組的名稱和選擇性描述，並選取一個使用者群組做為父群組。 選擇 [下一步]

3.  在 [定義成員資格準則]  頁面上，指定您是要包含父群組的所有成員，還是要先建立一個空群組。  您可以接著依據在 [Office 365 系統管理中心](http://go.microsoft.com/fwlink/?LinkId=698854)手動設定或從本機 Active Directory 同步處理的使用者**安全性群組**，包含或排除成員。 如果安全性群組的成員資格發生變更，以該安全性群組為依據的使用者群組可能也會變更。

    > [!IMPORTANT]
    > 目前，如果您的群組包含特定安全性或管理員群組的成員，而且您也排除特定群組的成員，則會移除您一開始所包含的成員。 若要建立同時具有包含成員和排除成員的群組，建議您先建立具有包含成員的父群組，再建立列有排除成員之群組的子系。 您可以接著在適用於 Intune 原則、設定檔和應用程式發佈的情況下使用該子群組。

    > [!NOTE]
    > 在 Azure 管理入口網站中，您可以建立以使用者回報的經理為基礎的群組。 此群組會是互動的，並隨著 Azure Active Directory 中該經理的團隊新增或移除員工而改變。 根據管理員建立 Azure 群組的程序會在名為**設定群組為「經理」群組**之一節中的[使用屬性來建立進階規則](https://azure.microsoft.com/en-us/documentation/articles/active-directory-accessmanagement-groups-with-advanced-rules/)部分中描述


4.  在 [定義直接成員資格]  頁面上，按一下 [瀏覽] 以包含或排除您指定的個別使用者。 如果您使用此選項選取的使用者不在指定的父群組中，這些使用者會自動新增到父群組。 在 [選取成員] 對話方塊的底部，您會發現手動新增使用者的選項。 如果您想要新增還沒有已註冊裝置的使用者，這會很有幫助。


5.  在 [摘要] 頁面上，檢閱將要執行的動作。 選擇 [完成]

您可以在 [群組] 工作區之父群組下的 [群組] 清單中，找到新建立的群組。 您也可以從這裡編輯或刪除該群組。

> [!TIP]
> 安全性群組是用來填入使用者群組的絕佳資源。 由於安全性群組會定義誰可存取哪些資源，因此，也能將它轉譯為 Intune 使用者群組。 從 Active Directory 同步處理至 Azure Active Directory 的安全性群組，或者透過 Office 365 系統管理中心或 Azure 系統管理入口網站直接在 Azure Active Directory 中建立的安全性群組，都可供您用來在 Intune 中建立使用者群組。

## 自訂系統管理員角色的檢視
已篩選的群組檢視可讓您自訂系統管理員可以根據其角色看到的檢視，並限制每位 IT 系統管理員可以管理的群組。 在下列情況時，這可能十分有用：

-   您的 IT 系統管理員應該只能將項目部署到特定使用者和裝置。

-   您只想為每位 IT 系統管理員顯示相關的群組。

您可以在 Intune 管理主控台中設定服務系統管理員的已篩選群組檢視。 如需詳細資訊，請參閱[設定啟動 Microsoft Intune 前的須知事項](/intune/get-started/what-to-know-before-you-start-microsoft-intune)

設定服務系統管理員的已篩選群組檢視之後，該系統管理員：

-   只能檢視和選取您在部署軟體或原則時，或在使用報表時所指定的群組

-   不會收到管理主控台的下列頁面上的狀態資訊：

    -   **系統概觀**

    -   **群組概觀**

    -   **Endpoint Protection 概觀**

    -   **警示概觀**

    -   **軟體概觀**

    -   **原則概觀**

### 設定已篩選的群組檢視

1.  在 Intune 管理主控台中，選擇 [系統管理]  &gt;  [系統管理員管理]  &gt;  [服務系統管理員]。

2.  選取您要篩選群組的服務系統管理員，然後按一下 [管理群組]

3.  在 [選取此服務系統管理員將看到的群組] 對話方塊中，新增所選取服務系統管理員可以存取的群組，然後按一下 [確定]。

設定已篩選的群組檢視之後，IT 系統管理員只能檢視和選取您選取的群組。

## 管理群組
在您建立群組之後，您將繼續根據組織需求來管理它們。

您可以編輯群組，來變更其名稱和說明，以及隸屬於該群組的成員。

您可以刪除不再符合組織需求的群組。 刪除群組不會刪除隸屬於該群組的使用者。

## 後續步驟

### 檢查您的設計
在您設定群組和原則之後，請藉由檢閱**預定的值**和**狀態**來檢查設計的實際影響

1. 從裝置群組中選取任何裝置，然後完整瀏覽畫面頂端的資訊類別。
2. 選取 [原則]  。 您會看到與下列螢幕擷取畫面類似的 Android 裝置原則設定。

![範例 iOS 設定原則](../media/Intune-Device-Policy-v.2.jpg)

每個原則都具有 [預定的值]  和 [狀態] 。 預定的值是您在指派原則時想要達到的值。 狀態則是您在一併考慮要套用到裝置的所有原則，以及硬體和作業系統的限制與需求時，實際達成的情況。  在螢幕擷取畫面中，您可以看到兩個明確的範例：

-   [允許簡單密碼] 已設為 [是]，如 [預定的值] 欄中所示，但其 [狀態] 為 [不適用]。 這是因為 Android 裝置不支援簡單的密碼。

-   同樣地，擴充的原則項目 [iOS 裝置的電子郵件設定] 不會套用到此裝置，因為它是 Android 裝置。

> 請記住，當有兩項不同限制等級的原則套用至同一部裝置或使用者時，實際上會套用限制更嚴格的原則。
