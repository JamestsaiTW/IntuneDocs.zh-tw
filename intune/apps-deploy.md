# <a name="how-to-assign-apps-to-groups-with-microsoft-intune"></a>如何使用 Microsoft Intune 將應用程式指派給群組

[!INCLUDE[azure_portal](./includes/azure_portal.md)]

將應用程式新增至 Intune 之後，可以將它指派給使用者和裝置。

不論應用程式是否由 Intune 管理，皆可指派到裝置。 使用下表有助於您了解將應用程式指派到使用者及裝置的各種選項︰

||||
|-|-|-|-|
|&nbsp;|註冊有 Intune 的裝置|未註冊有 Intune 的裝置|
|指派給使用者|是|是|
|指派給裝置|是|否|
|指派經包裝之應用程式或應用程式，與 Intune SDK (適用於應用程式保護原則) 合併運作|是|是|
|將應用程式指派為可用|是|是|
|將應用程式指派為必要項目|是|否|
|解除安裝應用程式|是|否|
|從 Intune 接收應用程式更新|是|否|
|使用者從公司入口網站應用程式安裝可用的應用程式|是|否|
|使用者從 Web 架構入口網站安裝可用的應用程式|是|是|

> [!NOTE]
> 您目前可指派 iOS 與 Android 應用程式 (企業營運與市集購買兩者) 未註冊有 Intune 的裝置。<br></br><br></br>
> 若要在未向 Intune 註冊的裝置上接收應用程式更新，裝置使用者必須巡覽至其公司入口網站，並手動安裝應用程式更新。

## <a name="how-to-assign-an-app"></a>如何指派應用程式

1. 登入 [Azure 入口網站](https://portal.azure.com)。
2. 選擇 [All services] (所有服務) > [Intune]。 Intune 位於 [監視 + 管理] 區段。
3. 在 [Intune] 刀鋒視窗上，選擇 [行動應用程式]。
1. 在**行動應用程式**工作負載中選擇 [管理]  >  [應用程式]。
2. 在應用程式刀鋒視窗清單中，按一下想要指派的應用程式。
3. 在 [概觀] 刀鋒視窗上，選擇 [管理] > [指派]。
4. 選擇 [新增群組]，然後在 [新增群組] 刀鋒視窗上選擇要或不要指派應用程式的 Azure AD 群組。
5. 為每個您選擇的應用程式，從以下項目選擇應用程式的 [指派類型]︰
    - **可透過已註冊裝置使用** - 使用者從公司入口網站應用程式或網站中安裝應用程式。
    - **是否有註冊皆可用** - 將此應用程式指派到其裝置未註冊有 Intune 的使用者群組。
    - **必要** - 此應用程式安裝所選群組中的裝置上。
    - **解除安裝** - 此應用程式會從所選群組中的裝置上解除安裝。
6. **僅限 iOS 應用程式** - 如果您已建立包含個別應用程式 VPN 設定的 iOS VPN 設定檔，您可以在 [VPN] 下加以選取。 應用程式執行時，即會開啟 VPN 連線。 如需詳細資訊，請參閱 [iOS 裝置的 VPN 設定](vpn-settings-ios.md)。
6. 完成之後，請選擇 [確定]，然後選擇 [儲存]。

應用程式現已指派給您所選取的群組。

## <a name="how-conflicts-between-app-intents-are-resolved"></a>如何解決應用程式用途之間的衝突

有時候，相同的應用程式會指派給多個群組，但使用不同的用途。 在這些情況下，使用此表格來了解產生的用途。

||||
|-|-|-|
|群組 1 用途|群組 2 用途|產生的用途|
|使用者為必要|使用者可用|必要且可用|
|使用者為必要|使用者無法使用|必要|
|使用者為必要|使用者解除安裝|必要|
|使用者可用|使用者無法使用|無法使用|
|使用者可用|使用者解除安裝|解除安裝|
|使用者無法使用|使用者解除安裝|解除安裝
|使用者為必要|裝置為必要|同時存在，閘道處理為必要 
|使用者為必要|裝置解除安裝|同時存在，閘道解析為必要 
|使用者可用|裝置為必要|同時存在，閘道解析為必要 (必要且可用)
|使用者可用|裝置解除安裝|同時存在，閘道解析可用。<br>應用程式會顯示在公司入口網站中。<br>如果已安裝應用程式 (作為具有先前用途的必要應用程式)，則會解除安裝該應用程式。<br>不過，如果使用者按一下從公司入口網站安裝，則會安裝應用程式，而不接受解除安裝用途。|
|使用者無法使用|裝置為必要|必要|
|使用者無法使用|裝置解除安裝|解除安裝|
|使用者解除安裝|裝置為必要|同時存在，閘道解析為必要|
|使用者解除安裝|裝置解除安裝|同時存在，閘道解析解除安裝|
|裝置為必要|裝置解除安裝|必要|
|使用者為必要且可用|使用者可用|必要且可用|
|使用者為必要且可用|使用者解除安裝|必要且可用|
|使用者為必要且可用|使用者無法使用|必要且可用|
|使用者為必要且可用|裝置為必要|同時存在，必要且可用
|使用者為必要且可用|裝置無法使用|必要且可用|
|使用者為必要且可用|裝置解除安裝|同時存在，閘道解析為必要。 必要 + 可用
|使用者無法使用|裝置無法使用|無法使用|
|使用者可用|裝置無法使用|可用|
|使用者為必要|裝置無法使用|必要|
|使用者可用而無需註冊|使用者為必要且可用|必要且可用
|使用者可用而無需註冊|使用者為必要|必要
|使用者可用而無需註冊|使用者無法使用|無法使用
|使用者可用而無需註冊|使用者可用|可用|
|使用者可用而無需註冊|裝置為必要|必要且可用而無需註冊|
|使用者可用而無需註冊|裝置無法使用|可用而無需註冊|
|使用者可用而無需註冊|裝置解除安裝|解除安裝且可用而無需註冊。<br>如果使用者未從公司入口網站安裝應用程式，則接受解除安裝。<br>如果使用者從公司入口網站安裝應用程式，則安裝將優先於解除安裝。|

>[!NOTE]
>僅針對受管理的 iOS 市集應用程式，當您將這些應用程式新增到 Intune 並指派為「必要」時，系統會自動將它們建立成包含「必要」及「可用」用途。
