# <a name="troubleshoot-app-installation-issues"></a>針對應用程式安裝問題進行疑難排解

在受 Microsoft Intune MDM 管理的裝置上，應用程式安裝有時可能會失敗。 當這些應用程式安裝失敗時，使用者可能無法輕易地了解失敗的原因，或是對問題進行疑難排解。 Microsoft Intune 提供應用程式安裝失敗詳細資料，可讓技術支援人員和 Intune 系統管理員檢視應用程式資訊，以處理使用者的協助要求。 Intune 中的 [疑難排解] 窗格會提供失敗的詳細資訊，包括使用者裝置上的受控應用程式相關詳細資料。 有關應用程式完整生命週期的詳細資訊，會在 [受控應用程式] 窗格中的每個個別裝置下提供。 您可以檢視安裝問題，例如當應用程式建立、修改、設定目標，以及傳遞至裝置時的問題。 

## <a name="to-review-app-troubleshooting-details"></a>檢視應用程式疑難排解詳細資料

Intune 會根據特定使用者裝置上安裝的應用程式，提供應用程式疑難排解的詳細資料。

1. 登入 [Azure 入口網站](https://portal.azure.com)。
2. 選擇 [All services] (所有服務) > [Intune]。 Intune 位於 [Monitoring + Management] (監視 + 管理) 區段。
3. 在 [Intune] 窗格上，選擇 [疑難排解]。
4. 按一下 [選取使用者] 來選取使用者以進行疑難排解。 [選取使用者] 窗格隨即顯示。
5. 鍵入名稱或電子郵件地址來選取使用者。 按一下窗格底部的 [選取]。 在 [疑難排解] 窗格中，會顯示使用者的疑難排解資訊。 
6. 從 [裝置] 清單中選取要進行疑難排解的裝置。
    ![[Intune 疑難排解] 窗格。](media/troubleshoot-app-install-01.png)
7. 從 [選取的裝置] 窗格中選取 [受控應用程式]。 受控應用程式清單隨即出現。
    ![由 Intune 所管理特定裝置的詳細資料。](media/troubleshoot-app-install-02.png)
8. 從 [安裝狀態] 指示失敗的清單中選取應用程式。
    ![顯示安裝失敗詳細資料的已選取應用程式。](media/troubleshoot-app-install-03.png)

    > [!Note]  
    > 可以將相同的應用程式指派給多個群組，但對於該應用程式有不同的預期動作 (用途)。 例如，如果在應用程式指派期間針對某個使用者排除了該應用程式，則應用程式已解決的用途會顯示 [已排除]。 如需詳細資訊，請參閱[如何解決應用程式用途之間的衝突](apps-deploy.md#how-conflicts-between-app-intents-are-resolved)。<br><br>
    > 如果必要的應用程式安裝失敗，您或您的技術服務人員將能夠同步裝置並重試應用程式安裝。

應用程式安裝錯誤詳細資料會指出此問題。 您可以使用這些詳細資料，以決定解決問題所要採取的最佳動作。 如需有關針對應用程式安裝問題進行疑難排解的詳細資訊，請參閱[針對應用程式安裝問題進行疑難排解的錯誤碼](https://blogs.technet.microsoft.com/intunesupport/2018/05/15/error-codes-for-troubleshooting-app-installation-issues) \(英文\)。

> [!Note]  
> 您也可以將瀏覽器指向 [https://aka.ms/intunetroubleshooting](https://aka.ms/intunetroubleshooting)來存取 [疑難排解] 窗格。

## <a name="app-installation-errors"></a>應用程式安裝錯誤

下列錯誤訊息和描述提供 Android 和 iOS 安裝錯誤的相關詳細資料。 

### <a name="android-errors"></a>Android 錯誤

|    錯誤訊息/錯誤碼    |    說明    |
|----------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|    應用程式無法安裝。 (0xC7D14FB5)    |    Intune 無法判斷 Android 應用程式安裝錯誤的根本原因時，就會顯示此錯誤訊息。 Android 在失敗期間不會提供任何資訊。       當 APK 下載成功，但應用程式安裝失敗時，就會傳回此錯誤。 發生此錯誤的較常見原因是 APK 檔案損毀，無法安裝在裝置上。 一個可能原因是 Google Play Protect 基於安全性考量而封鎖應用程式安裝。 此錯誤的另一個可能原因是裝置不支援應用程式。 例如，如果應用程式需要 API 21+ 版，而裝置目前有 API 19 版。         針對 DA 和 KNOX 裝置，Intune 會傳回此錯誤，雖然可能會出現通知，指出使用者可以按一下進行重試，但如果問題出在 APK，則不會一直失敗。 如果應用程式是可用的應用程式，則可以關閉通知。 不過，如果需要應用程式，則無法將它關閉。        |
|    因為安裝 (APK) 檔案下載後於安裝之前刪除，所以應用程式安裝已取消。 (0xC7D14FBA)    |    APK 下載成功，但使用者安裝應用程式之前，已從裝置移除檔案。 如果下載到安裝的間隔時間太長，就會發生此問題。 例如，使用者已取消原始安裝、等候一段時間，然後按一下通知再試一次。         此錯誤訊息只會針對 DA 案例傳回。 KNOX 案例會透過無訊息方式完成。 但會顯示重試通知，讓使用者可以接受而不是取消。 如果應用程式是可用的應用程式，則可以關閉通知。 不過，如果需要應用程式，則無法將它關閉。    |
|    因為在安裝期間重新啟動處理序，所以應用程式的安裝已取消。 (0xC7D14FBB)    |    裝置在 APK 安裝過程中已重新開機，而導致取消安裝。        此錯誤訊息會針對 DA 和 KNOX 裝置傳回。 Intune 顯示使用者可以按一下以重試的通知。 如果應用程式是可用的應用程式，則可以關閉通知。 不過，如果需要應用程式，則無法將它關閉。    |
|    在完成順利安裝此應用程式後，卻未偵測到該應用程式。 (0x87D1041C)    |    使用者已明確解除安裝應用程式。 此錯誤不是從用戶端傳回。 當應用程式在某個時間點安裝，但之後使用者將它解除安裝時，就會產生此錯誤。 只有必要的應用程式才會出現此錯誤。 使用者可以解除安裝非必要的應用程式。 此錯誤只會發生在 DA 中。 KNOX 會封鎖解除安裝受控應用程式。       下一次同步會在裝置上重新張貼通知，讓使用者進行安裝。   使用者可以忽略通知。 此錯誤會繼續回報，直到使用者安裝應用程式為止。    |
|    因為發生意外錯誤，所以下載失敗。 (0xC7D14FB2)    |    當下載失敗時，就會發生此錯誤。 發生此錯誤的常見原因是 Wi-Fi 問題或連線速度緩慢。       此錯誤只會針對 DA 案例傳回。 針對 KNOX 案例，系統不會提示使用者安裝，這會透過無訊息方式完成。 Intune 顯示使用者可以按一下以重試的通知。 如果應用程式是可用的應用程式，則可以關閉通知。 不過，如果需要應用程式，則無法將它關閉。    |
|    因為發生意外錯誤，所以下載失敗。 將會在下次裝置同步時重試原則。 (0xC7D15078)    |    當下載失敗時，就會發生此錯誤。 發生此錯誤的常見原因是 Wi-Fi 問題或連線速度緩慢。       此錯誤只會針對 DA 案例傳回。 針對 KNOX 案例，系統不會提示使用者安裝，這會透過無訊息方式完成。    |
|    終端使用者已取消應用程式安裝。 (0xC7D14FB1)    |    使用者已明確解除安裝應用程式。 當使用者已取消 Android OS 安裝活動時，就會傳回此錯誤。 使用者已在 OS 安裝提示出現時按下 [取消] 按鈕，或已在提示畫面以外的位置按一下。        此錯誤只會針對 DA 案例傳回。 針對 KNOX 案例，系統不會提示使用者安裝，這會透過無訊息方式完成。 Intune 顯示使用者可以按一下以重試的通知。 如果應用程式是可用的應用程式，則可以關閉通知。 不過，如果需要應用程式，則無法將它關閉。    |
|    檔案下載程序意外停止。 (0xC7D15015)    |    OS 在下載程序完成之前就已將它停止。 當裝置電力偏低或下載時間太長時，就會發生此錯誤。       此錯誤只會針對 DA 案例傳回。 針對 KNOX 案例，系統不會提示使用者安裝，這會透過無訊息方式完成。 Intune 顯示使用者可以按一下以重試的通知。 如果應用程式是可用的應用程式，則可以關閉通知。 不過，如果需要應用程式，則無法將它關閉。    |
|    檔案下載服務意外停止。 將會在下次裝置同步時重試原則。 (0xC7D1507C)    |    OS 在下載程序完成之前就已將它停止。 當裝置電力偏低或下載時間太長時，就會發生此錯誤。       此錯誤只會針對 DA 案例傳回。 針對 KNOX 案例，系統不會提示使用者安裝，這會透過無訊息方式完成。    |
